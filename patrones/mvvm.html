MVVM (Model-View-ViewModel)

Separar la UI de la lógica de presentación mediante un ViewModel que expone estado observable, permitiendo data-binding bidireccional.

Arquitectura

📖 Descripción

MVVM extiende MVC introduciendo el ViewModel: un objeto que expone propiedades observables y comandos. La Vista se enlaza directamente a esas propiedades mediante data-binding (sin código manual de actualización). El ViewModel no conoce la Vista. Es el patrón dominante en frameworks reactivos como Angular, Vue, SwiftUI y WPF.

🎯 Casos de uso prácticos

1. Formulario de búsqueda reactivo

El usuario escribe en el campo de búsqueda; el ViewModel filtra resultados automáticamente mediante binding.

2. Dashboard de métricas en tiempo real

El ViewModel recibe datos del servidor y actualiza propiedades observables; la Vista se refresca automáticamente.

💻 Ejemplos de código

JavaScript
// Simulación simple de reactividad (sin framework)
function reactivo(obj, onChange) {
  return new Proxy(obj, {
    set(target, prop, value) {
      target[prop] = value;
      onChange(prop, value);
      return true;
    }
  });
}

// ViewModel
class BusquedaViewModel {
  constructor(productos) {
    this._productos = productos;
    this.estado = reactivo(
      { termino: '', resultados: productos, cargando: false },
      (prop, val) => this._render(prop, val)
    );
  }

  buscar(termino) {
    this.estado.cargando = true;
    // Simula async
    setTimeout(() => {
      this.estado.resultados = this._productos.filter(p =>
        p.nombre.toLowerCase().includes(termino.toLowerCase())
      );
      this.estado.cargando = false;
    }, 300);
    this.estado.termino = termino;
  }

  _render(prop, val) {
    if (prop === 'resultados') {
      console.log(`Vista: ${val.length} resultado(s):`);
      val.forEach(r => console.log(` - ${r.nombre}`));
    }
    if (prop === 'cargando') console.log(`Vista: ${val ? 'cargando...' : 'listo'}`);
  }
}

const vm = new BusquedaViewModel([
  { nombre: 'Laptop Pro' }, { nombre: 'Laptop Air' },
  { nombre: 'Teclado Mecánico' }, { nombre: 'Mouse Inalámbrico' }
]);

vm.buscar('laptop');
Java
// ViewModel con SimpleStringProperty (JavaFX style)
public class BusquedaViewModel {
    private ObservableList<Producto> resultados = FXCollections.observableArrayList();
    private StringProperty termino = new SimpleStringProperty("");
    private List<Producto> todos;

    public BusquedaViewModel(List<Producto> productos) {
        this.todos = productos;
        // Binding: cuando cambia termino, filtrar resultados
        termino.addListener((obs, old, nuevo) -> filtrar(nuevo));
    }

    private void filtrar(String t) {
        resultados.setAll(todos.stream()
            .filter(p -> p.getNombre().toLowerCase().contains(t.toLowerCase()))
            .collect(Collectors.toList()));
    }

    // La Vista hace binding a estos
    public StringProperty terminoProperty() { return termino; }
    public ObservableList<Producto> getResultados() { return resultados; }
}

🛠️ Software que lo implementa

  • Angular (con RxJS)
  • Vue.js (reactive data)
  • React + MobX
  • WPF / Xamarin (.NET)
  • SwiftUI
  • Android Jetpack (ViewModel + LiveData)

🚫 Cuándo NO usar

  • Para interfaces simples sin estado reactivo; el overhead del binding no compensa.
  • En sistemas donde el data-binding bidireccional puede causar bucles infinitos de actualización.
  • Con frameworks que no soportan binding nativo; implementarlo manualmente es costoso.
