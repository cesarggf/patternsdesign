Memento

Capturar y externalizar el estado interno de un objeto sin violar su encapsulamiento, para poder restaurarlo más tarde.

Comportamiento

📖 Descripción

Memento define tres roles: el Originador (objeto cuyo estado se guarda), el Memento (cápsula que almacena el estado) y el Caretaker (gestiona el historial de mementos). El Originador crea y restaura mementos; el Caretaker los almacena sin acceder a su contenido.

🎯 Casos de uso prácticos

1. Editor con historial de cambios

Guardar snapshots del contenido del editor para implementar múltiples niveles de undo.

2. Checkpoints en un videojuego

Guardar el estado completo del juego (posición, inventario, puntos) en puntos de control.

💻 Ejemplos de código

JavaScript
// Memento
class EstadoEditor {
  constructor(contenido) { this.contenido = contenido; }
  obtenerContenido() { return this.contenido; }
}

// Originador
class Editor {
  constructor() { this.contenido = ''; }
  escribir(texto) { this.contenido += texto; }
  guardar()    { return new EstadoEditor(this.contenido); }
  restaurar(memento) { this.contenido = memento.obtenerContenido(); }
  mostrar()    { console.log(`Contenido: "${this.contenido}"`); }
}

// Caretaker
class HistorialEditor {
  constructor() { this.estados = []; }
  guardar(editor) { this.estados.push(editor.guardar()); }
  deshacer(editor) {
    if (this.estados.length > 1) {
      this.estados.pop(); // descarta el actual
      editor.restaurar(this.estados[this.estados.length - 1]);
    }
  }
}

const editor    = new Editor();
const historial = new HistorialEditor();

historial.guardar(editor);
editor.escribir('Hola ');
historial.guardar(editor);
editor.escribir('mundo');
historial.guardar(editor);
editor.mostrar(); // Hola mundo

historial.deshacer(editor);
editor.mostrar(); // Hola
Java
public class EstadoEditor {
    private final String contenido;
    public EstadoEditor(String c) { contenido = c; }
    public String obtener() { return contenido; }
}

public class Editor {
    private String contenido = "";
    public void escribir(String t) { contenido += t; }
    public EstadoEditor guardar() { return new EstadoEditor(contenido); }
    public void restaurar(EstadoEditor m) { contenido = m.obtener(); }
    public String getContenido() { return contenido; }
}

public class HistorialEditor {
    private Deque<EstadoEditor> estados = new ArrayDeque<>();

    public void guardar(Editor e) { estados.push(e.guardar()); }
    public void deshacer(Editor e) {
        if (estados.size() > 1) {
            estados.pop();
            e.restaurar(estados.peek());
        }
    }
}

🚫 Cuándo NO usar

  • Cuando el estado del objeto es muy grande, guardar copias frecuentes puede consumir demasiada memoria.
  • Si el historial no es una funcionalidad requerida, el patrón agrega complejidad innecesaria.
  • Para estados que contienen referencias a objetos externos, la captura puede ser incompleta.
