Chain of Responsibility

Pasar una solicitud a lo largo de una cadena de manejadores, donde cada uno decide procesarla o pasarla al siguiente.

Comportamiento

📖 Descripción

Chain of Responsibility desacopla al emisor de una solicitud de sus receptores. Cada manejador tiene una referencia al siguiente en la cadena. Al recibir una solicitud, el manejador la procesa si puede, o la pasa al siguiente. Esto permite componer cadenas de procesamiento flexibles y modificarlas en tiempo de ejecución.

🎯 Casos de uso prácticos

1. Pipeline de validación de formulario

Validar presencia, formato email y longitud mínima de contraseña en cadena.

2. Sistema de soporte técnico

Nivel 1 (básico) → Nivel 2 (técnico) → Nivel 3 (experto) según la complejidad del ticket.

💻 Ejemplos de código

JavaScript
class Manejador {
  constructor(nombre) { this.nombre = nombre; this.siguiente = null; }
  setSiguiente(m) { this.siguiente = m; return m; }
  manejar(solicitud) {
    if (this.siguiente) return this.siguiente.manejar(solicitud);
    console.log(`Sin manejador para: ${solicitud}`);
  }
}

class ValidadorPresencia extends Manejador {
  manejar(datos) {
    if (!datos.email || !datos.password) {
      return console.log('❌ Campos requeridos vacíos');
    }
    super.manejar(datos);
  }
}
class ValidadorEmail extends Manejador {
  manejar(datos) {
    if (!/\S+@\S+\.\S+/.test(datos.email)) {
      return console.log('❌ Email inválido');
    }
    super.manejar(datos);
  }
}
class ValidadorPassword extends Manejador {
  manejar(datos) {
    if (datos.password.length < 8) {
      return console.log('❌ Contraseña muy corta (mín. 8 chars)');
    }
    console.log('✅ Validación completa');
  }
}

const v1 = new ValidadorPresencia('presencia');
const v2 = new ValidadorEmail('email');
const v3 = new ValidadorPassword('password');
v1.setSiguiente(v2).setSiguiente(v3);

v1.manejar({ email: 'test@mail.com', password: '12345678' }); // ✅
v1.manejar({ email: 'mal-email', password: '12345678' });     // ❌
Java
public abstract class Manejador {
    protected Manejador siguiente;
    public Manejador setSiguiente(Manejador m) { siguiente = m; return m; }
    public abstract void manejar(Map<String, String> datos);
}

public class ValidadorPresencia extends Manejador {
    public void manejar(Map<String, String> datos) {
        if (datos.get("email") == null || datos.get("password") == null) {
            System.out.println("❌ Campos requeridos vacíos");
            return;
        }
        if (siguiente != null) siguiente.manejar(datos);
    }
}

public class ValidadorEmail extends Manejador {
    public void manejar(Map<String, String> datos) {
        if (!datos.get("email").contains("@")) {
            System.out.println("❌ Email inválido"); return;
        }
        if (siguiente != null) siguiente.manejar(datos);
    }
}

public class ValidadorPassword extends Manejador {
    public void manejar(Map<String, String> datos) {
        if (datos.get("password").length() < 8) {
            System.out.println("❌ Contraseña muy corta"); return;
        }
        System.out.println("✅ Validación completa");
    }
}

🚫 Cuándo NO usar

  • Cuando hay un único manejador posible; no tiene sentido armar la cadena.
  • Si ningún manejador puede garantizar que la solicitud será procesada, la solicitud se puede perder silenciosamente.
  • Cadenas muy largas degradan el rendimiento por el recorrido secuencial.

🛠️ Software que lo implementa

  • Java Servlet Filters
  • Spring Security FilterChain
  • Express.js / Koa middleware
  • Apache Camel (pipeline de procesamiento)
  • Node.js EventEmitter
