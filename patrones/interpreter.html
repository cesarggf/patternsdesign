Interpreter

Dada una gramática para un lenguaje, definir una representación de esa gramática junto con un intérprete que use dicha representación para interpretar sentencias en el lenguaje.

Comportamiento

📖 Descripción

Interpreter define una clase por cada regla gramatical del lenguaje. Las expresiones terminales son las hojas; las no terminales combinan otras expresiones. Útil para lenguajes pequeños o DSLs (Domain Specific Languages) como consultas de búsqueda, expresiones matemáticas o reglas de negocio.

🎯 Casos de uso prácticos

1. Calculadora de expresiones

Interpretar expresiones matemáticas como '3 + 4 * 2' siguiendo reglas de gramática.

2. Motor de reglas de negocio

Evaluar reglas como 'edad > 18 AND suscripcion = premium' sobre un contexto de datos.

💻 Ejemplos de código

JavaScript
// Expresiones terminales
class Numero {
  constructor(valor) { this.valor = valor; }
  interpretar() { return this.valor; }
}

// Expresiones no terminales
class Suma {
  constructor(izq, der) { this.izq = izq; this.der = der; }
  interpretar() { return this.izq.interpretar() + this.der.interpretar(); }
}
class Multiplicacion {
  constructor(izq, der) { this.izq = izq; this.der = der; }
  interpretar() { return this.izq.interpretar() * this.der.interpretar(); }
}
class Resta {
  constructor(izq, der) { this.izq = izq; this.der = der; }
  interpretar() { return this.izq.interpretar() - this.der.interpretar(); }
}

// Evaluar: (3 + 4) * 2 - 1
const expresion = new Resta(
  new Multiplicacion(
    new Suma(new Numero(3), new Numero(4)),
    new Numero(2)
  ),
  new Numero(1)
);

console.log('Resultado:', expresion.interpretar()); // 13
Java
public interface Expresion {
    int interpretar();
}

public class Numero implements Expresion {
    private int valor;
    public Numero(int v) { valor = v; }
    public int interpretar() { return valor; }
}

public class Suma implements Expresion {
    private Expresion izq, der;
    public Suma(Expresion i, Expresion d) { izq=i; der=d; }
    public int interpretar() { return izq.interpretar() + der.interpretar(); }
}

public class Multiplicacion implements Expresion {
    private Expresion izq, der;
    public Multiplicacion(Expresion i, Expresion d) { izq=i; der=d; }
    public int interpretar() { return izq.interpretar() * der.interpretar(); }
}

// Evaluar (3 + 4) * 2
public class Main {
    public static void main(String[] args) {
        Expresion exp = new Multiplicacion(
            new Suma(new Numero(3), new Numero(4)),
            new Numero(2)
        );
        System.out.println("Resultado: " + exp.interpretar()); // 14
    }
}

🚫 Cuándo NO usar

  • Para gramáticas complejas con muchas reglas; la cantidad de clases crece exponencialmente.
  • Cuando el rendimiento es crítico; el patrón no es eficiente para gramáticas grandes.
  • Existen herramientas como ANTLR o parser combinators más adecuadas para lenguajes reales.
