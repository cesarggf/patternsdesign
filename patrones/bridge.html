Bridge

Desacoplar una abstracción de su implementación para que ambas puedan variar independientemente.

Estructural

📖 Descripción

Bridge separa una jerarquía de abstracciones de una jerarquía de implementaciones mediante composición en lugar de herencia. La abstracción mantiene una referencia a un objeto implementador y le delega el trabajo concreto. Esto evita la explosión combinatorial de clases cuando ambas dimensiones pueden variar.

🎯 Casos de uso prácticos

1. Renderizado multiplataforma

Formas geométricas (Círculo, Rectángulo) separadas del motor de renderizado (Canvas, SVG).

2. Sistemas de notificaciones

Tipos de alerta (Urgente, Informativa) separados del canal de envío (Email, SMS, Push).

💻 Ejemplos de código

JavaScript
// Implementador
class Renderizador {
  renderizarCirculo(x, y, radio) { throw new Error('Abstracto'); }
}
class RenderizadorCanvas extends Renderizador {
  renderizarCirculo(x, y, radio) {
    console.log(`Canvas: círculo en (${x},${y}) r=${radio}`);
  }
}
class RenderizadorSVG extends Renderizador {
  renderizarCirculo(x, y, radio) {
    console.log(`<circle cx="${x}" cy="${y}" r="${radio}"/>`);
  }
}

// Abstracción
class Forma {
  constructor(renderizador) { this.renderizador = renderizador; }
  dibujar() { throw new Error('Abstracto'); }
}
class Circulo extends Forma {
  constructor(x, y, radio, renderizador) {
    super(renderizador);
    this.x = x; this.y = y; this.radio = radio;
  }
  dibujar() { this.renderizador.renderizarCirculo(this.x, this.y, this.radio); }
}

// Uso — misma forma, distintos renderizadores
const c1 = new Circulo(10, 20, 15, new RenderizadorCanvas());
const c2 = new Circulo(10, 20, 15, new RenderizadorSVG());
c1.dibujar(); c2.dibujar();
Java
public interface Renderizador {
    void renderizarCirculo(int x, int y, int radio);
}

public class RenderizadorCanvas implements Renderizador {
    public void renderizarCirculo(int x, int y, int radio) {
        System.out.println("Canvas: círculo en (" + x + "," + y + ") r=" + radio);
    }
}

public abstract class Forma {
    protected Renderizador renderizador;
    public Forma(Renderizador r) { this.renderizador = r; }
    public abstract void dibujar();
}

public class Circulo extends Forma {
    private int x, y, radio;
    public Circulo(int x, int y, int radio, Renderizador r) {
        super(r); this.x = x; this.y = y; this.radio = radio;
    }
    public void dibujar() { renderizador.renderizarCirculo(x, y, radio); }
}

🚫 Cuándo NO usar

  • Cuando solo hay una dimensión de variación; un simple polimorfismo es suficiente.
  • Si la abstracción y la implementación raramente cambian independientemente, el patrón añade complejidad sin beneficio.
  • Para sistemas pequeños donde la jerarquía de clases no crecerá.
