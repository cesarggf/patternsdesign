Template Method

Definir el esqueleto de un algoritmo en una operación, dejando que las subclases redefinan ciertos pasos sin cambiar la estructura del algoritmo.

Comportamiento

📖 Descripción

Template Method implementa el esqueleto de un proceso en la clase base. Los pasos invariantes se implementan en la base; los pasos variables se declaran abstractos para que las subclases los completen. Evita duplicación de código cuando múltiples clases siguen el mismo proceso pero con variaciones en pasos específicos.

🎯 Casos de uso prácticos

1. Proceso de exportación de datos

El proceso siempre es: abrir → leer datos → transformar → escribir → cerrar. El formato (CSV, XML, JSON) varía.

2. Ciclo de vida de tests

setUp → ejecutarTest → tearDown es el esqueleto; cada test implementa su propio cuerpo.

💻 Ejemplos de código

JavaScript
// Clase base con template method
class ExportadorDatos {
  // Template method — no se sobreescribe
  exportar(datos) {
    const encabezado = this.generarEncabezado();
    const cuerpo = datos.map(fila => this.formatearFila(fila)).join(this.separador());
    const pie = this.generarPie();
    console.log([encabezado, cuerpo, pie].filter(Boolean).join('\n'));
  }

  // Pasos variables — las subclases los implementan
  generarEncabezado() { return ''; }
  generarPie()        { return ''; }
  separador()         { return '\n'; }
  formatearFila(fila) { throw new Error('Implementar'); }
}

class ExportadorCSV extends ExportadorDatos {
  generarEncabezado() { return 'nombre,edad,ciudad'; }
  formatearFila(f)    { return `${f.nombre},${f.edad},${f.ciudad}`; }
}

class ExportadorJSON extends ExportadorDatos {
  generarEncabezado() { return '['; }
  generarPie()        { return ']'; }
  separador()         { return ',\n'; }
  formatearFila(f)    { return JSON.stringify(f); }
}

const datos = [
  { nombre: 'Cesar', edad: 30, ciudad: 'Madrid' },
  { nombre: 'Ana',   edad: 25, ciudad: 'Bogotá'  }
];

new ExportadorCSV().exportar(datos);
new ExportadorJSON().exportar(datos);
Java
public abstract class ExportadorDatos {
    // Template method
    public final void exportar(List<Map<String,String>> datos) {
        System.out.println(generarEncabezado());
        datos.forEach(f -> System.out.println(formatearFila(f)));
        System.out.println(generarPie());
    }

    protected String generarEncabezado() { return ""; }
    protected String generarPie()        { return ""; }
    protected abstract String formatearFila(Map<String,String> fila);
}

public class ExportadorCSV extends ExportadorDatos {
    protected String generarEncabezado() { return "nombre,edad,ciudad"; }
    protected String formatearFila(Map<String,String> f) {
        return f.get("nombre")+","+f.get("edad")+","+f.get("ciudad");
    }
}

public class ExportadorXML extends ExportadorDatos {
    protected String generarEncabezado() { return "<datos>"; }
    protected String generarPie()        { return "</datos>"; }
    protected String formatearFila(Map<String,String> f) {
        return "  <fila nombre=\""+f.get("nombre")+"\" edad=\""+f.get("edad")+"\"/>";
    }
}

🚫 Cuándo NO usar

  • Cuando la variación entre subclases es tan grande que el esqueleto común es mínimo.
  • Si se necesita cambiar el algoritmo en tiempo de ejecución; Strategy es más apropiado.
  • Puede violar el Principio de Sustitución de Liskov si las subclases cambian drásticamente el comportamiento.
