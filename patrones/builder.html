Builder

Separar la construcción de un objeto complejo de su representación para crear distintas representaciones con el mismo proceso.

Creacional

📖 Descripción

Builder permite construir objetos paso a paso. El Director controla el orden de construcción; el Builder concreto implementa cada paso. El resultado final se obtiene al final del proceso. Es ideal cuando un objeto requiere muchos parámetros opcionales o pasos de configuración.

🎯 Casos de uso prácticos

1. Constructor de consultas SQL

Armar dinámicamente consultas SELECT con cláusulas WHERE, ORDER BY y LIMIT opcionales.

2. Generador de reportes PDF

Un reporte puede incluir encabezado, gráficas, tablas y pie de página en distintas combinaciones.

💻 Ejemplos de código

JavaScript
class ConsultaSQL {
  constructor() {
    this.tabla = ''; this.condiciones = [];
    this.columnas = ['*']; this.limite = null;
  }
}

class ConstructorConsulta {
  constructor() { this.consulta = new ConsultaSQL(); }

  desde(tabla)       { this.consulta.tabla = tabla;             return this; }
  seleccionar(...cols){ this.consulta.columnas = cols;          return this; }
  donde(condicion)   { this.consulta.condiciones.push(condicion); return this; }
  limitar(n)         { this.consulta.limite = n;                return this; }

  construir() {
    let sql = `SELECT ${this.consulta.columnas.join(', ')} FROM ${this.consulta.tabla}`;
    if (this.consulta.condiciones.length)
      sql += ` WHERE ${this.consulta.condiciones.join(' AND ')}`;
    if (this.consulta.limite)
      sql += ` LIMIT ${this.consulta.limite}`;
    return sql;
  }
}

const sql = new ConstructorConsulta()
  .desde('usuarios')
  .seleccionar('id', 'nombre', 'email')
  .donde('activo = true')
  .donde('edad > 18')
  .limitar(10)
  .construir();

console.log(sql);
// SELECT id, nombre, email FROM usuarios WHERE activo = true AND edad > 18 LIMIT 10
Java
public class ConsultaSQL {
    private String tabla;
    private List<String> columnas = new ArrayList<>();
    private List<String> condiciones = new ArrayList<>();
    private Integer limite;

    // Getters omitidos por brevedad
    @Override
    public String toString() {
        String cols = columnas.isEmpty() ? "*" : String.join(", ", columnas);
        StringBuilder sb = new StringBuilder("SELECT " + cols + " FROM " + tabla);
        if (!condiciones.isEmpty())
            sb.append(" WHERE ").append(String.join(" AND ", condiciones));
        if (limite != null)
            sb.append(" LIMIT ").append(limite);
        return sb.toString();
    }

    public static class Builder {
        private ConsultaSQL consulta = new ConsultaSQL();

        public Builder desde(String tabla) { consulta.tabla = tabla; return this; }
        public Builder seleccionar(String... cols) {
            consulta.columnas.addAll(Arrays.asList(cols)); return this;
        }
        public Builder donde(String cond) { consulta.condiciones.add(cond); return this; }
        public Builder limitar(int n) { consulta.limite = n; return this; }
        public ConsultaSQL construir() { return consulta; }
    }

    public static void main(String[] args) {
        ConsultaSQL sql = new ConsultaSQL.Builder()
            .desde("usuarios")
            .seleccionar("id", "nombre", "email")
            .donde("activo = true")
            .limitar(10)
            .construir();
        System.out.println(sql);
    }
}

🚫 Cuándo NO usar

  • Para objetos simples con pocos atributos; un constructor con parámetros es más directo.
  • Cuando la construcción siempre sigue exactamente el mismo proceso sin variaciones.
  • Si el objeto no es inmutable ni complejo, el patrón agrega burocracia sin beneficio.
