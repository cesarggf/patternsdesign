State

Permitir a un objeto alterar su comportamiento cuando su estado interno cambia, pareciendo que cambia de clase.

Comportamiento

📖 Descripción

State encapsula cada estado posible en un objeto separado. El contexto mantiene una referencia al estado actual y delega las operaciones en él. Cuando ocurre una transición, el contexto cambia su referencia de estado. Elimina grandes condicionales if/switch basados en estado.

🎯 Casos de uso prácticos

1. Reproductor multimedia

Estados: Detenido, Reproduciendo, Pausado. Cada uno maneja Play/Pause/Stop de forma diferente.

2. Semáforo

Estados: Rojo, Amarillo, Verde. Cada estado define la duración y el siguiente estado.

💻 Ejemplos de código

JavaScript
class Reproductor {
  constructor() { this.estado = new EstadoDetenido(this); }
  cambiarEstado(estado) { this.estado = estado; }
  play()  { this.estado.play();  }
  pause() { this.estado.pause(); }
  stop()  { this.estado.stop();  }
}

class EstadoDetenido {
  constructor(r) { this.r = r; }
  play()  { console.log('▶ Iniciando...'); this.r.cambiarEstado(new EstadoReproduciendo(this.r)); }
  pause() { console.log('⚠️ No está reproduciendo'); }
  stop()  { console.log('⚠️ Ya está detenido'); }
}

class EstadoReproduciendo {
  constructor(r) { this.r = r; }
  play()  { console.log('⚠️ Ya está reproduciendo'); }
  pause() { console.log('⏸ Pausado'); this.r.cambiarEstado(new EstadoPausado(this.r)); }
  stop()  { console.log('⏹ Detenido'); this.r.cambiarEstado(new EstadoDetenido(this.r)); }
}

class EstadoPausado {
  constructor(r) { this.r = r; }
  play()  { console.log('▶ Reanudando...'); this.r.cambiarEstado(new EstadoReproduciendo(this.r)); }
  pause() { console.log('⚠️ Ya está pausado'); }
  stop()  { console.log('⏹ Detenido'); this.r.cambiarEstado(new EstadoDetenido(this.r)); }
}

const p = new Reproductor();
p.play(); p.pause(); p.play(); p.stop();
Java
public interface EstadoReproductor {
    void play();
    void pause();
    void stop();
}

public class Reproductor {
    private EstadoReproductor estado;
    public Reproductor() { estado = new EstadoDetenido(this); }
    public void cambiarEstado(EstadoReproductor e) { estado = e; }
    public void play()  { estado.play(); }
    public void pause() { estado.pause(); }
    public void stop()  { estado.stop(); }
}

public class EstadoDetenido implements EstadoReproductor {
    private Reproductor r;
    public EstadoDetenido(Reproductor r) { this.r = r; }
    public void play()  { System.out.println("▶ Iniciando"); r.cambiarEstado(new EstadoReproduciendo(r)); }
    public void pause() { System.out.println("⚠️ No está reproduciendo"); }
    public void stop()  { System.out.println("⚠️ Ya detenido"); }
}

public class EstadoReproduciendo implements EstadoReproductor {
    private Reproductor r;
    public EstadoReproduciendo(Reproductor r) { this.r = r; }
    public void play()  { System.out.println("⚠️ Ya reproduciendo"); }
    public void pause() { System.out.println("⏸ Pausado"); r.cambiarEstado(new EstadoPausado(r)); }
    public void stop()  { System.out.println("⏹ Detenido"); r.cambiarEstado(new EstadoDetenido(r)); }
}

🚫 Cuándo NO usar

  • Cuando solo hay dos estados y la lógica es trivial; un boolean es más claro.
  • Si las transiciones de estado son pocas y simples, los condicionales son más legibles.
  • Demasiados estados sin relación entre sí pueden producir una explosión de clases.
