Iterator

Proporcionar una forma de acceder secuencialmente a los elementos de una colección sin exponer su representación interna.

Comportamiento

📖 Descripción

Iterator encapsula la lógica de recorrido de una colección en un objeto separado. El cliente usa la interfaz del iterador sin conocer si la colección es un array, árbol, grafo u otra estructura. Esto permite múltiples recorridos simultáneos e independientes sobre la misma colección.

🎯 Casos de uso prácticos

1. Recorrido de árbol binario

Proveer iteradores para recorrido en orden, preorden y postorden sin cambiar la clase árbol.

2. Playlist de música

Un iterador que recorre canciones en orden, aleatorio o cíclico sin cambiar la colección.

💻 Ejemplos de código

JavaScript
class Playlist {
  constructor() { this.canciones = []; }
  agregar(cancion) { this.canciones.push(cancion); }

  // Iterador en orden
  [Symbol.iterator]() {
    let indice = 0;
    const canciones = this.canciones;
    return {
      next() {
        return indice < canciones.length
          ? { value: canciones[indice++], done: false }
          : { done: true };
      }
    };
  }

  // Iterador aleatorio
  *iteradorAleatorio() {
    const copia = [...this.canciones].sort(() => Math.random() - 0.5);
    for (const cancion of copia) yield cancion;
  }
}

const lista = new Playlist();
lista.agregar('Bohemian Rhapsody');
lista.agregar('Hotel California');
lista.agregar('Stairway to Heaven');

console.log('--- Orden normal ---');
for (const c of lista) console.log(c);

console.log('--- Aleatorio ---');
for (const c of lista.iteradorAleatorio()) console.log(c);
Java
public class Playlist implements Iterable<String> {
    private List<String> canciones = new ArrayList<>();

    public void agregar(String cancion) { canciones.add(cancion); }

    @Override
    public Iterator<String> iterator() {
        return canciones.iterator(); // iterador en orden
    }

    public Iterator<String> iteradorAleatorio() {
        List<String> copia = new ArrayList<>(canciones);
        Collections.shuffle(copia);
        return copia.iterator();
    }

    public static void main(String[] args) {
        Playlist lista = new Playlist();
        lista.agregar("Bohemian Rhapsody");
        lista.agregar("Hotel California");

        System.out.println("--- Orden normal ---");
        for (String c : lista) System.out.println(c);

        System.out.println("--- Aleatorio ---");
        lista.iteradorAleatorio().forEachRemaining(System.out::println);
    }
}

🚫 Cuándo NO usar

  • Para arrays simples donde un for/forEach es más claro y directo.
  • Cuando la colección siempre se recorre de una sola manera y no hay variación.
  • En lenguajes modernos donde los iterables/generators ya cubren este patrón de forma nativa.

🛠️ Software que lo implementa

  • Java java.util.Iterator / Iterable
  • Python generators (yield)
  • C# IEnumerable / foreach
  • JavaScript Symbol.iterator / for...of
  • MongoDB Cursor
