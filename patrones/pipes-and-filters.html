Pipes and Filters

Descomponer el procesamiento de mensajes en una secuencia de pasos independientes (filtros) conectados por canales (pipes).

Integración

📖 Descripción

Pipes and Filters construye un pipeline donde cada filtro realiza una transformación concreta sobre el mensaje y pasa el resultado al siguiente mediante un pipe. Cada filtro es independiente, reutilizable y puede probarse de forma aislada. El pipeline puede reconfigurarse cambiando el orden de los filtros o añadiendo nuevos.

🎯 Casos de uso prácticos

1. Pipeline ETL

Extraer datos CSV → limpiar nulos → normalizar fechas → transformar moneda → cargar en BD.

2. Procesamiento de imágenes

Recibir imagen → redimensionar → convertir a grises → aplicar marca de agua → comprimir → guardar.

💻 Ejemplos de código

JavaScript
// Pipe: conecta filtros en cadena
class Pipeline {
  constructor() { this.filtros = []; }
  agregar(filtro) { this.filtros.push(filtro); return this; }
  procesar(entrada) {
    return this.filtros.reduce((dato, filtro) => filtro(dato), entrada);
  }
}

// Filtros (funciones puras)
const limpiarEspacios  = texto => texto.trim();
const aNombresCompletos = texto => texto.split(',').map(n => n.trim()).join(', ');
const aMayusculas      = texto => texto.toUpperCase();
const agregarPrefijo   = texto => `>> ${texto}`;

// Construcción del pipeline
const pipeline = new Pipeline()
  .agregar(limpiarEspacios)
  .agregar(aNombresCompletos)
  .agregar(aMayusculas)
  .agregar(agregarPrefijo);

const resultado = pipeline.procesar('  juan perez , ana garcia , luis mora  ');
console.log(resultado);
// >> JUAN PEREZ, ANA GARCIA, LUIS MORA
Java
import java.util.*;
import java.util.function.*;

public class Pipeline<T> {
    private List<UnaryOperator<T>> filtros = new ArrayList<>();

    public Pipeline<T> agregar(UnaryOperator<T> filtro) {
        filtros.add(filtro); return this;
    }
    public T procesar(T entrada) {
        T resultado = entrada;
        for (UnaryOperator<T> f : filtros) resultado = f.apply(resultado);
        return resultado;
    }

    public static void main(String[] args) {
        Pipeline<String> pipeline = new Pipeline<String>()
            .agregar(String::trim)
            .agregar(s -> s.replaceAll("\\s+", " "))
            .agregar(String::toUpperCase)
            .agregar(s -> ">> " + s);

        System.out.println(pipeline.procesar("  hola   mundo  "));
        // >> HOLA MUNDO
    }
}

🛠️ Software que lo implementa

  • Apache Camel
  • Spring Integration
  • Apache NiFi
  • Unix shell pipes
  • Node.js Transform Streams
  • Logstash (pipeline de logs)

🚫 Cuándo NO usar

  • Cuando los pasos tienen dependencias de estado complejas entre ellos; el modelo stateless de los filtros no aplica.
  • Si la latencia acumulada de cada filtro es inaceptable para el caso de uso.
  • Para lógica de decisión compleja con muchas ramas; usa Router en su lugar.
