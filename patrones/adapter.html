Adapter

Convertir la interfaz de una clase en otra que el cliente espera, permitiendo que clases incompatibles trabajen juntas.

Estructural

📖 Descripción

Adapter actúa como un traductor entre dos interfaces incompatibles. Envuelve un objeto existente (adaptee) y expone la interfaz que el cliente necesita, delegando las llamadas al objeto envuelto con las transformaciones necesarias. Existen dos variantes: Adapter de objeto (composición) y Adapter de clase (herencia múltiple).

🎯 Casos de uso prácticos

1. Integración con API externa

Tu sistema espera una interfaz `IPago` pero la librería de terceros tiene métodos distintos.

2. Migración de almacenamiento

Adaptar un repositorio legacy con interfaz distinta para que funcione con el nuevo contrato.

💻 Ejemplos de código

JavaScript
// Interfaz esperada por el cliente
class IPago {
  pagar(monto) { throw new Error('Abstracto'); }
}

// Clase existente con interfaz incompatible (librería externa)
class PayPalSDK {
  realizarTransaccion(cantidad, moneda) {
    console.log(`PayPal: ${cantidad} ${moneda} procesados`);
  }
}

// Adapter
class AdapterPayPal extends IPago {
  constructor() {
    super();
    this.paypal = new PayPalSDK();
  }
  pagar(monto) {
    // Traduce la llamada
    this.paypal.realizarTransaccion(monto, 'EUR');
  }
}

// Cliente usa siempre IPago
function procesarCompra(pasarela, monto) {
  pasarela.pagar(monto);
}

procesarCompra(new AdapterPayPal(), 99.99);
Java
// Interfaz objetivo
public interface IPago {
    void pagar(double monto);
}

// Clase incompatible (librería externa)
public class PayPalSDK {
    public void realizarTransaccion(double cantidad, String moneda) {
        System.out.println("PayPal: " + cantidad + " " + moneda);
    }
}

// Adapter
public class AdapterPayPal implements IPago {
    private PayPalSDK paypal = new PayPalSDK();

    @Override
    public void pagar(double monto) {
        paypal.realizarTransaccion(monto, "EUR");
    }
}

// Cliente
public class Tienda {
    public void procesarCompra(IPago pasarela, double monto) {
        pasarela.pagar(monto);
    }
    public static void main(String[] args) {
        new Tienda().procesarCompra(new AdapterPayPal(), 99.99);
    }
}

🚫 Cuándo NO usar

  • Cuando puedes modificar la clase original directamente sin romper otros contratos.
  • Si la adaptación requiere cambios muy profundos en el comportamiento, considera Facade o Decorator.
  • Demasiados Adapters en cadena dificultan el rastreo de errores y el debugging.

🛠️ Software que lo implementa

  • Java InputStreamReader (adapta InputStream a Reader)
  • Spring HandlerAdapter (MVC)
  • SLF4J (adapta Log4j, JUL, Logback)
  • Jackson ObjectMapper (adapta JSON a POJO)
  • Vue.js / React adapters de componentes
