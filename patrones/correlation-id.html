Correlation ID

Asignar un identificador único a cada flujo de procesamiento para poder rastrear y correlacionar mensajes relacionados a través de múltiples sistemas.

Integración

📖 Descripción

Cuando un mensaje pasa por múltiples servicios, el Correlation ID viaja en el encabezado de cada mensaje o petición. Cada servicio propaga el ID a sus llamadas salientes. Esto permite agrupar todos los logs, trazas y eventos de una transacción completa, siendo la base del distributed tracing.

🎯 Casos de uso prácticos

1. Trazabilidad de pedidos

Un pedido genera un Correlation ID que viaja por los servicios de pago, inventario y envío, permitiendo ver el historial completo en los logs.

2. Debugging de errores en producción

Un usuario reporta un error; con su Correlation ID se recuperan todos los logs relacionados de todos los microservicios en segundos.

💻 Ejemplos de código

JavaScript
const { randomUUID } = require('crypto');

// Middleware que inyecta/propaga Correlation ID
function correlationMiddleware(req, res, next) {
  req.correlationId = req.headers['x-correlation-id'] || randomUUID();
  res.setHeader('x-correlation-id', req.correlationId);
  next();
}

// Logger con Correlation ID
class Logger {
  constructor(correlationId) { this.id = correlationId; }
  info(msg)  { console.log(`[${this.id}] INFO  ${msg}`); }
  error(msg) { console.error(`[${this.id}] ERROR ${msg}`); }
}

// Servicio que propaga el ID
async function procesarPedido(pedido, correlationId) {
  const log = new Logger(correlationId);
  log.info(`Iniciando procesamiento del pedido ${pedido.id}`);

  // Simula llamada a otro servicio pasando el ID
  await llamarServicioPago(pedido, correlationId);
  await llamarServicioInventario(pedido, correlationId);
  log.info(`Pedido ${pedido.id} completado`);
}

async function llamarServicioPago(pedido, correlationId) {
  const log = new Logger(correlationId);
  log.info(`[ServicioPago] Cobrando €${pedido.monto}`);
}
async function llamarServicioInventario(pedido, correlationId) {
  const log = new Logger(correlationId);
  log.info(`[Inventario] Reservando ${pedido.producto}`);
}

// Uso
const id = randomUUID();
procesarPedido({ id: 42, monto: 99.99, producto: 'Laptop' }, id);
Java
import org.slf4j.MDC;
import java.util.UUID;

public class CorrelationFilter implements Filter {
    public static final String HEADER = "X-Correlation-ID";

    @Override
    public void doFilter(ServletRequest req, ServletResponse res, FilterChain chain)
            throws IOException, ServletException {
        HttpServletRequest httpReq = (HttpServletRequest) req;
        String correlationId = httpReq.getHeader(HEADER);
        if (correlationId == null) correlationId = UUID.randomUUID().toString();

        // SLF4J MDC propaga el ID en todos los logs del hilo
        MDC.put("correlationId", correlationId);
        ((HttpServletResponse) res).setHeader(HEADER, correlationId);
        try {
            chain.doFilter(req, res);
        } finally {
            MDC.clear();
        }
    }
}
// En logback.xml: %X{correlationId} en el pattern

🚫 Cuándo NO usar

  • En sistemas de un único servicio donde el rastreo distribuido no aplica.
  • Cuando ya usas un framework de tracing completo (OpenTelemetry, Jaeger) que gestiona los IDs automáticamente.
  • Para operaciones fire-and-forget donde el seguimiento no tiene valor.
