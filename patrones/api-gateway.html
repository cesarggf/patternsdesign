API Gateway

Proveer un punto de entrada único para todos los clientes que acceden a los servicios backend, centralizando corte transversal como autenticación, rate limiting, logging y enrutamiento.

Integración

📖 Descripción

API Gateway actúa como proxy inverso inteligente delante de los microservicios. Recibe todas las peticiones externas, aplica políticas globales (auth, throttling, transformación) y las redirige al servicio correspondiente. Evita que los clientes conozcan la topología interna y simplifica la gestión de la API pública.

🎯 Casos de uso prácticos

1. Plataforma móvil + web

App móvil y SPA web llaman al mismo gateway; este enruta a user-service, product-service y order-service.

2. Agregación de respuestas (BFF)

El gateway llama a tres microservicios en paralelo y combina la respuesta en un solo payload para el cliente.

💻 Ejemplos de código

JavaScript
const express = require('express');
// Simulación — sin require real de http-proxy

class ApiGateway {
  constructor() {
    this.app = express();
    this.rutas = {};
    this.middlewares = [];
  }

  usarMiddleware(fn) { this.middlewares.push(fn); return this; }

  registrar(prefijo, servicioUrl) {
    this.rutas[prefijo] = servicioUrl; return this;
  }

  iniciar(puerto) {
    this.app.use(express.json());

    // Aplicar middlewares globales
    this.middlewares.forEach(mw => this.app.use(mw));

    // Enrutamiento dinámico
    this.app.use((req, res) => {
      const prefijo = Object.keys(this.rutas).find(p => req.path.startsWith(p));
      if (!prefijo) return res.status(404).json({ error: 'Servicio no encontrado' });
      console.log(`[Gateway] ${req.method} ${req.path} → ${this.rutas[prefijo]}`);
      res.json({ proxied_to: this.rutas[prefijo], path: req.path });
    });

    this.app.listen(puerto, () => console.log(`Gateway en :${puerto}`));
  }
}

// Middlewares de corte transversal
const autenticar = (req, res, next) => {
  if (!req.headers.authorization) return res.status(401).json({ error: 'No autorizado' });
  next();
};
const rateLimiter = (() => {
  const contadores = {};
  return (req, res, next) => {
    const ip = req.ip;
    contadores[ip] = (contadores[ip] || 0) + 1;
    if (contadores[ip] > 100) return res.status(429).json({ error: 'Too many requests' });
    next();
  };
})();

const gw = new ApiGateway()
  .usarMiddleware(autenticar)
  .usarMiddleware(rateLimiter)
  .registrar('/usuarios',  'http://user-service:3001')
  .registrar('/productos', 'http://product-service:3002')
  .registrar('/pedidos',   'http://order-service:3003');

gw.iniciar(8080);
Java
@RestController
@RequestMapping("/api")
public class ApiGatewayController {
    private Map<String, String> rutas = Map.of(
        "/usuarios",  "http://user-service:3001",
        "/productos", "http://product-service:3002",
        "/pedidos",   "http://order-service:3003"
    );

    private final RestTemplate restTemplate = new RestTemplate();

    @RequestMapping("/**")
    public ResponseEntity<?> proxy(HttpServletRequest req) {
        String path = req.getRequestURI().replaceFirst("/api", "");
        String prefijo = rutas.keySet().stream()
            .filter(path::startsWith).findFirst().orElse(null);
        if (prefijo == null) return ResponseEntity.notFound().build();

        String destino = rutas.get(prefijo) + path;
        System.out.println("[Gateway] → " + destino);
        return restTemplate.getForEntity(destino, String.class);
    }
}

🚫 Cuándo NO usar

  • Cuando solo tienes un monolito o un par de servicios; añade latencia y complejidad sin beneficio.
  • Si el gateway se convierte en un cuello de botella único sin alta disponibilidad.
  • Para comunicación interna entre microservicios; usa service mesh (Istio, Linkerd).
