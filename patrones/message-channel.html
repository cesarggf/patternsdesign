Message Channel

Conectar dos aplicaciones mediante un canal de mensajes que desacopla al emisor del receptor.

Integración

📖 Descripción

Message Channel define un conducto lógico por el que fluyen mensajes entre un productor y uno o más consumidores. El productor publica sin conocer quién consume; el consumidor lee sin conocer quién produce. Los canales pueden ser point-to-point (un solo consumidor) o publish-subscribe (múltiples consumidores). Es el bloque fundamental de toda arquitectura de mensajería.

🎯 Casos de uso prácticos

1. Procesamiento de pedidos

El servicio web publica un pedido en un canal; el servicio de inventario y el de facturación lo consumen de forma independiente.

2. Ingesta de logs

Múltiples microservicios escriben en un canal de logs; un consumidor centralizado los indexa en Elasticsearch.

💻 Ejemplos de código

JavaScript
const { EventEmitter } = require('events');

// Canal point-to-point simple
class Canal extends EventEmitter {
  enviar(mensaje) {
    console.log(`[Canal] → publicando: ${JSON.stringify(mensaje)}`);
    this.emit('mensaje', mensaje);
  }
}

// Productor
class ServicioPedidos {
  constructor(canal) { this.canal = canal; }
  crearPedido(pedido) {
    console.log(`[Pedidos] Creando pedido #${pedido.id}`);
    this.canal.enviar({ tipo: 'PEDIDO_CREADO', payload: pedido });
  }
}

// Consumidores
const canal = new Canal();
canal.on('mensaje', msg => {
  if (msg.tipo === 'PEDIDO_CREADO') {
    console.log(`[Inventario] Reservando stock para pedido #${msg.payload.id}`);
  }
});
canal.on('mensaje', msg => {
  if (msg.tipo === 'PEDIDO_CREADO') {
    console.log(`[Facturación] Generando factura para pedido #${msg.payload.id}`);
  }
});

const svc = new ServicioPedidos(canal);
svc.crearPedido({ id: 42, producto: 'Laptop', qty: 1 });
Java
import java.util.*;

// Canal genérico
public class Canal<T> {
    private List<Consumer<T>> suscriptores = new ArrayList<>();

    public void suscribir(Consumer<T> s) { suscriptores.add(s); }
    public void enviar(T mensaje) {
        System.out.println("[Canal] publicando: " + mensaje);
        suscriptores.forEach(s -> s.accept(mensaje));
    }
}

// Uso
public class Main {
    public static void main(String[] args) {
        Canal<Map<String,Object>> canal = new Canal<>();

        // Suscriptores (consumidores)
        canal.suscribir(m -> System.out.println("[Inventario] Reservando stock: " + m));
        canal.suscribir(m -> System.out.println("[Facturación] Generando factura: " + m));

        // Productor
        Map<String,Object> pedido = Map.of("id", 42, "producto", "Laptop");
        canal.enviar(pedido);
    }
}

🛠️ Software que lo implementa

  • Apache Kafka (topics)
  • RabbitMQ (exchanges + queues)
  • AWS SQS / SNS
  • Azure Service Bus
  • Google Cloud Pub/Sub
  • ActiveMQ

🚫 Cuándo NO usar

  • Cuando la comunicación es estrictamente síncrona y requiere respuesta inmediata; usa HTTP/gRPC.
  • Para volúmenes extremadamente bajos sin necesidad de desacoplamiento temporal.
  • Si el orden de los mensajes es crítico y el broker no lo garantiza.
