Prototype

Crear nuevos objetos copiando una instancia existente (prototipo) sin depender de su clase concreta.

Creacional

📖 Descripción

Prototype delega la creación de copias al propio objeto mediante un método `clonar()`. Permite duplicar objetos costosos de inicializar (como los que requieren operaciones de red o disco) sin repetir esa inicialización. Existen dos variantes: copia superficial (shallow) y copia profunda (deep).

🎯 Casos de uso prácticos

1. Plantillas de documentos

Clonar un documento base ya formateado para crear variantes sin repetir la inicialización.

2. Objetos de juego

En un videojuego, clonar enemigos ya configurados en lugar de crear cada uno desde cero.

💻 Ejemplos de código

JavaScript
class Personaje {
  constructor(nombre, nivel, habilidades) {
    this.nombre = nombre;
    this.nivel = nivel;
    this.habilidades = [...habilidades]; // copia profunda del array
  }

  clonar() {
    return new Personaje(this.nombre, this.nivel, this.habilidades);
  }

  toString() {
    return `${this.nombre} [Nv.${this.nivel}] — ${this.habilidades.join(', ')}`;
  }
}

// Prototipo base
const guerrero = new Personaje('Guerrero', 1, ['espada', 'escudo']);

// Clonamos y personalizamos
const guerrero2 = guerrero.clonar();
guerrero2.nombre = 'Guerrero Élite';
guerrero2.nivel  = 5;
guerrero2.habilidades.push('magia');

console.log(guerrero.toString());  // Guerrero [Nv.1] — espada, escudo
console.log(guerrero2.toString()); // Guerrero Élite [Nv.5] — espada, escudo, magia
Java
public class Personaje implements Cloneable {
    private String nombre;
    private int nivel;
    private List<String> habilidades;

    public Personaje(String nombre, int nivel, List<String> habilidades) {
        this.nombre = nombre;
        this.nivel = nivel;
        this.habilidades = new ArrayList<>(habilidades);
    }

    @Override
    public Personaje clone() {
        try {
            Personaje clon = (Personaje) super.clone();
            clon.habilidades = new ArrayList<>(this.habilidades); // deep copy
            return clon;
        } catch (CloneNotSupportedException e) {
            throw new RuntimeException(e);
        }
    }

    public static void main(String[] args) {
        Personaje base = new Personaje("Guerrero", 1,
            Arrays.asList("espada", "escudo"));

        Personaje elite = base.clone();
        elite.nombre = "Guerrero Élite";
        elite.nivel  = 5;
        elite.habilidades.add("magia");

        System.out.println(base);   // Guerrero Nv.1
        System.out.println(elite);  // Guerrero Élite Nv.5
    }
}

🚫 Cuándo NO usar

  • Cuando los objetos tienen pocas propiedades simples; un constructor es más claro.
  • Si el objeto contiene referencias circulares, la clonación profunda puede ser extremadamente compleja.
  • Cuando clonar viola invariantes del objeto (ej: IDs únicos que no deben duplicarse).
