Flyweight

Usar compartición para soportar eficientemente grandes cantidades de objetos de grano fino.

Estructural

📖 Descripción

Flyweight reduce el uso de memoria separando el estado intrínseco (compartido, inmutable) del estado extrínseco (único por instancia, gestionado por el cliente). La fábrica Flyweight garantiza que los objetos con el mismo estado intrínseco se compartan en lugar de duplicarse.

🎯 Casos de uso prácticos

1. Caracteres en un editor de texto

Millones de caracteres comparten el objeto de fuente/estilo; solo la posición es única.

2. Partículas en un motor de juego

Miles de partículas de fuego comparten la textura y el color; solo posición y velocidad varían.

💻 Ejemplos de código

JavaScript
// Flyweight — estado intrínseco
class TipoArbol {
  constructor(nombre, color, textura) {
    this.nombre   = nombre;
    this.color    = color;
    this.textura  = textura;
  }
  dibujar(x, y) {
    console.log(`🌳 ${this.nombre} (${this.color}) en (${x},${y})`);
  }
}

// Fábrica Flyweight
class FabricaArboles {
  constructor() { this.tipos = {}; }

  obtener(nombre, color, textura) {
    const clave = `${nombre}-${color}`;
    if (!this.tipos[clave]) {
      this.tipos[clave] = new TipoArbol(nombre, color, textura);
      console.log(`Creando nuevo tipo: ${clave}`);
    }
    return this.tipos[clave];
  }
  get total() { return Object.keys(this.tipos).length; }
}

// Bosque con 1000 árboles pero pocos tipos
const fabrica = new FabricaArboles();
const bosque = [];

for (let i = 0; i < 500; i++) {
  bosque.push({ tipo: fabrica.obtener('Roble','verde','rugosa'), x: i, y: i*2 });
  bosque.push({ tipo: fabrica.obtener('Pino','verde oscuro','lisa'), x: i*3, y: i });
}

bosque[0].tipo.dibujar(bosque[0].x, bosque[0].y);
console.log(`Tipos de árbol: ${fabrica.total}, Árboles: ${bosque.length}`);
Java
public class TipoArbol {
    private String nombre, color, textura;
    public TipoArbol(String n, String c, String t) {
        nombre=n; color=c; textura=t;
    }
    public void dibujar(int x, int y) {
        System.out.println("🌳 "+nombre+" ("+color+") en ("+x+","+y+")");
    }
}

public class FabricaArboles {
    private static Map<String, TipoArbol> tipos = new HashMap<>();

    public static TipoArbol obtener(String nombre, String color, String textura) {
        String clave = nombre + "-" + color;
        tipos.computeIfAbsent(clave, k -> new TipoArbol(nombre, color, textura));
        return tipos.get(clave);
    }
    public static int totalTipos() { return tipos.size(); }
}

// Árbol en el bosque (estado extrínseco)
public class Arbol {
    private int x, y;
    private TipoArbol tipo;
    public Arbol(int x, int y, TipoArbol tipo) { this.x=x; this.y=y; this.tipo=tipo; }
    public void dibujar() { tipo.dibujar(x, y); }
}

🚫 Cuándo NO usar

  • Cuando el número de objetos es pequeño; la complejidad del patrón no compensa.
  • Si el estado extrínseco es difícil de externalizar o complica significativamente el código cliente.
  • Cuando la memoria no es una restricción real del sistema.
