Message Translator

Transformar el formato o estructura de un mensaje para que sea compatible con el sistema destino.

Integración

📖 Descripción

Message Translator convierte mensajes entre diferentes formatos o esquemas de datos (XML↔JSON, SOAP↔REST, modelo de dominio A↔modelo de dominio B). Actúa como un mapeador en el pipeline de integración, permitiendo que sistemas heterogéneos se comuniquen sin cambiar sus contratos internos.

🎯 Casos de uso prácticos

1. Integración CRM ↔ ERP

El CRM envía clientes en formato propio; el traductor los convierte al esquema esperado por el ERP antes de sincronizar.

2. API legacy → API REST moderna

Transformar respuestas XML de un sistema legacy al JSON que espera la app móvil.

💻 Ejemplos de código

JavaScript
// Formato origen (sistema CRM externo)
const clienteCRM = {
  cliente_id: 'C-001',
  nombre_completo: 'Cesar García',
  correo_electronico: 'cesar@example.com',
  telefono_movil: '+34600000000',
  fecha_alta: '21/06/2026'
};

// Traductor → formato ERP interno
function traducirCRMaERP(crm) {
  const [dia, mes, anio] = crm.fecha_alta.split('/');
  return {
    id:        crm.cliente_id,
    fullName:  crm.nombre_completo,
    email:     crm.correo_electronico,
    phone:     crm.telefono_movil,
    createdAt: `${anio}-${mes}-${dia}T00:00:00Z`,
    source:    'CRM'
  };
}

const clienteERP = traducirCRMaERP(clienteCRM);
console.log(clienteERP);
Java
// DTO origen
public record ClienteCRM(String cliente_id, String nombre_completo,
    String correo_electronico, String telefono_movil, String fecha_alta) {}

// DTO destino
public record ClienteERP(String id, String fullName, String email,
    String phone, String createdAt, String source) {}

// Traductor
public class TraductorClienteCRM {
    public ClienteERP traducir(ClienteCRM crm) {
        String[] partes = crm.fecha_alta().split("/");
        String iso = partes[2]+"-"+partes[1]+"-"+partes[0]+"T00:00:00Z";
        return new ClienteERP(
            crm.cliente_id(), crm.nombre_completo(),
            crm.correo_electronico(), crm.telefono_movil(),
            iso, "CRM"
        );
    }
}

🛠️ Software que lo implementa

  • Apache Camel (Message Translator / Transformer)
  • MuleSoft DataWeave
  • AWS Glue (ETL)
  • Spring Integration Transformer
  • XSLT (XML→XML)
  • MapStruct (Java bean mapping)

🚫 Cuándo NO usar

  • Cuando ambos sistemas pueden acordar un formato común; la traducción añade latencia y puntos de fallo.
  • Si la transformación es tan compleja que requiere lógica de negocio; separa en un paso diferente.
  • Para transformaciones bidireccionales complejas que no son invertibles.
