Abstract Factory

Proveer una interfaz para crear familias de objetos relacionados sin especificar sus clases concretas.

Creacional

📖 Descripción

Abstract Factory agrupa fábricas relacionadas bajo una interfaz común. Cuando el sistema necesita un conjunto de objetos compatibles entre sí (por ejemplo, componentes UI para distintas plataformas), la fábrica abstracta garantiza que solo se instancien objetos de la misma familia.

🎯 Casos de uso prácticos

1. Temas de UI (claro / oscuro)

Una app crea botones y menús con el estilo coherente según el tema seleccionado.

2. Drivers de base de datos

El sistema instancia conexiones, comandos y readers específicos para MySQL o PostgreSQL.

💻 Ejemplos de código

JavaScript
// Familias de productos
class BotonClaro { render() { return '<button class="btn-light">OK</button>'; } }
class BotonOscuro { render() { return '<button class="btn-dark">OK</button>'; } }
class MenuClaro  { render() { return '<nav class="menu-light">…</nav>'; } }
class MenuOscuro { render() { return '<nav class="menu-dark">…</nav>'; } }

// Fábricas concretas
class FabricaClara {
  crearBoton() { return new BotonClaro(); }
  crearMenu()  { return new MenuClaro(); }
}
class FabricaOscura {
  crearBoton() { return new BotonOscuro(); }
  crearMenu()  { return new MenuOscuro(); }
}

// Cliente
function construirUI(fabrica) {
  const boton = fabrica.crearBoton();
  const menu  = fabrica.crearMenu();
  console.log(boton.render());
  console.log(menu.render());
}

construirUI(new FabricaClara());
construirUI(new FabricaOscura());
Java
public interface FabricaUI {
    Boton crearBoton();
    Menu crearMenu();
}

public class FabricaClara implements FabricaUI {
    public Boton crearBoton() { return new BotonClaro(); }
    public Menu  crearMenu()  { return new MenuClaro(); }
}
public class FabricaOscura implements FabricaUI {
    public Boton crearBoton() { return new BotonOscuro(); }
    public Menu  crearMenu()  { return new MenuOscuro(); }
}

// Cliente
public class App {
    private Boton boton;
    private Menu  menu;

    public App(FabricaUI fabrica) {
        boton = fabrica.crearBoton();
        menu  = fabrica.crearMenu();
    }
    public void render() {
        boton.render();
        menu.render();
    }
    public static void main(String[] args) {
        new App(new FabricaClara()).render();
        new App(new FabricaOscura()).render();
    }
}

🚫 Cuándo NO usar

  • Cuando solo tienes una familia de productos; basta con Factory Method.
  • Si agregar nuevos tipos de productos requiere cambiar la interfaz abstracta, viola el principio abierto/cerrado.
  • Para sistemas pequeños, la cantidad de clases introducida puede ser desproporcionada.

🛠️ Software que lo implementa

  • Java AWT/Swing (LookAndFeel)
  • Spring DataSource / DataSourceAutoConfiguration
  • JDBC API (Connection, Statement, ResultSet)
  • Apache Commons DBCP
  • SWT (Standard Widget Toolkit)
