Singleton

Garantizar que una clase tenga una única instancia y proporcionar un punto de acceso global a ella.

Creacional

📖 Descripción

Singleton restringe la creación de instancias de una clase a un solo objeto. Se logra ocultando el constructor y exponiendo un método estático que devuelve siempre la misma instancia almacenada internamente. Es útil para recursos compartidos como configuraciones, conexiones a base de datos o registros de log.

🎯 Casos de uso prácticos

1. Gestor de configuración global

Una aplicación lee un archivo de configuración una sola vez y todas las partes del sistema acceden a la misma instancia.

2. Pool de conexión a base de datos

Un sistema necesita reutilizar conexiones costosas a una base de datos desde distintos módulos.

💻 Ejemplos de código

JavaScript
class Configuracion {
  constructor() {
    if (Configuracion._instancia) {
      return Configuracion._instancia;
    }
    this.datos = { idioma: 'es', tema: 'oscuro', version: '2.0' };
    Configuracion._instancia = this;
  }

  obtener(clave) { return this.datos[clave]; }
  establecer(clave, valor) { this.datos[clave] = valor; }
}

// Uso
const config1 = new Configuracion();
const config2 = new Configuracion();

config1.establecer('tema', 'claro');
console.log(config2.obtener('tema')); // 'claro' - misma instancia
console.log(config1 === config2);     // true
Java
public class Configuracion {
    private static Configuracion instancia;
    private Map<String, String> datos = new HashMap<>();

    private Configuracion() {
        datos.put("idioma", "es");
        datos.put("tema", "oscuro");
        datos.put("version", "2.0");
    }

    public static synchronized Configuracion obtenerInstancia() {
        if (instancia == null) {
            instancia = new Configuracion();
        }
        return instancia;
    }

    public String obtener(String clave) { return datos.get(clave); }
    public void establecer(String clave, String valor) { datos.put(clave, valor); }

    // Uso
    public static void main(String[] args) {
        Configuracion c1 = Configuracion.obtenerInstancia();
        Configuracion c2 = Configuracion.obtenerInstancia();
        c1.establecer("tema", "claro");
        System.out.println(c2.obtener("tema")); // claro
        System.out.println(c1 == c2);           // true
    }
}

🚫 Cuándo NO usar

  • Cuando necesitas múltiples instancias en entornos de prueba (dificulta el mocking).
  • En aplicaciones multihilo sin sincronización adecuada puede causar condiciones de carrera.
  • Cuando la clase tiene estado mutable global que complica el razonamiento del sistema.
  • Si viola el principio de responsabilidad única (la clase gestiona su propio ciclo de vida).

🛠️ Software que lo implementa

  • Java Runtime (Runtime.getRuntime())
  • Spring ApplicationContext
  • Log4j / SLF4J Logger
  • Hibernate SessionFactory
  • Node.js módulos (module cache)
