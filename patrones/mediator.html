Mediator

Definir un objeto que encapsule cómo interactúan un conjunto de objetos, promoviendo el bajo acoplamiento.

Comportamiento

📖 Descripción

Mediator centraliza las comunicaciones entre objetos (colegas) para evitar que se referencien directamente. Cada colega solo conoce al mediador; cuando necesita comunicarse con otro, lo hace a través del mediador. Reduce las dependencias de n² a n, pero puede convertirse en un punto de fallo central.

🎯 Casos de uso prácticos

1. Chat grupal

Los usuarios no se envían mensajes directamente; todos pasan por el servidor de chat (mediador).

2. Sistema de control de tráfico aéreo

Las aeronaves no se comunican entre sí; toda coordinación pasa por la torre de control.

💻 Ejemplos de código

JavaScript
class SalaChat {
  constructor() { this.usuarios = []; }

  unirse(usuario) {
    this.usuarios.push(usuario);
    this.notificar(`${usuario.nombre} se unió al chat`, null);
  }

  enviarMensaje(mensaje, emisor) {
    this.usuarios
      .filter(u => u !== emisor)
      .forEach(u => u.recibirMensaje(`${emisor.nombre}: ${mensaje}`));
  }

  notificar(mensaje, excluir) {
    this.usuarios
      .filter(u => u !== excluir)
      .forEach(u => u.recibirMensaje(`[Sistema] ${mensaje}`));
  }
}

class Usuario {
  constructor(nombre, sala) {
    this.nombre = nombre;
    this.sala = sala;
    sala.unirse(this);
  }
  enviar(mensaje) { this.sala.enviarMensaje(mensaje, this); }
  recibirMensaje(msg) { console.log(`${this.nombre} recibe: ${msg}`); }
}

const sala  = new SalaChat();
const cesar = new Usuario('Cesar', sala);
const ana   = new Usuario('Ana', sala);
cesar.enviar('Hola a todos!');
Java
public interface Mediador {
    void enviarMensaje(String mensaje, Usuario emisor);
    void unirse(Usuario usuario);
}

public class SalaChat implements Mediador {
    private List<Usuario> usuarios = new ArrayList<>();

    public void unirse(Usuario u) { usuarios.add(u); }

    public void enviarMensaje(String mensaje, Usuario emisor) {
        usuarios.stream()
            .filter(u -> u != emisor)
            .forEach(u -> u.recibirMensaje(emisor.getNombre() + ": " + mensaje));
    }
}

public class Usuario {
    private String nombre;
    private Mediador sala;

    public Usuario(String nombre, Mediador sala) {
        this.nombre = nombre; this.sala = sala; sala.unirse(this);
    }
    public void enviar(String msg) { sala.enviarMensaje(msg, this); }
    public void recibirMensaje(String msg) { System.out.println(nombre+": "+msg); }
    public String getNombre() { return nombre; }
}

🚫 Cuándo NO usar

  • Si las interacciones entre objetos son pocas y simples; la indirección no aporta.
  • Cuando el mediador acumula demasiada lógica y se convierte en un objeto 'dios'.
  • Si los colegas necesitan comunicarse directamente por razones de rendimiento.

🛠️ Software que lo implementa

  • Spring ApplicationEventPublisher
  • MediatR (.NET)
  • Java MVC frameworks (DispatcherServlet)
  • Redux (store como mediador)
  • Node.js EventEmitter
