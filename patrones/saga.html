Saga

Gestionar transacciones distribuidas que abarcan múltiples servicios mediante una secuencia de transacciones locales con compensaciones en caso de fallo.

Integración

📖 Descripción

Saga divide una transacción larga en transacciones locales de cada servicio. Si un paso falla, ejecuta transacciones compensatorias para deshacer los pasos anteriores. Existen dos variantes: Choreography (cada servicio publica eventos y reacciona) y Orchestration (un orquestador central coordina los pasos).

🎯 Casos de uso prácticos

1. Reserva de viaje

Reservar vuelo → hotel → coche de alquiler. Si falla el coche, cancelar hotel y vuelo como compensación.

2. Proceso de compra e-commerce

Crear pedido → cobrar pago → reducir inventario → enviar. Si falla el cobro, cancelar el pedido.

💻 Ejemplos de código

JavaScript
class SagaOrquestador {
  constructor(pasos) { this.pasos = pasos; this.ejecutados = []; }

  async ejecutar(contexto) {
    for (const paso of this.pasos) {
      try {
        console.log(`▶ Ejecutando: ${paso.nombre}`);
        await paso.ejecutar(contexto);
        this.ejecutados.unshift(paso); // prepend para compensar en orden inverso
      } catch (err) {
        console.error(`❌ Falló: ${paso.nombre} — ${err.message}`);
        console.log('↩ Iniciando compensación...');
        await this.compensar(contexto);
        throw err;
      }
    }
    console.log('✅ Saga completada');
  }

  async compensar(contexto) {
    for (const paso of this.ejecutados) {
      console.log(`↩ Compensando: ${paso.nombre}`);
      await paso.compensar(contexto);
    }
  }
}

Java
public interface PasoSaga {
    void ejecutar(Map<String,Object> ctx) throws Exception;
    void compensar(Map<String,Object> ctx);
    String getNombre();
}

public class SagaOrquestador {
    private List<PasoSaga> pasos;
    private Deque<PasoSaga> ejecutados = new ArrayDeque<>();

    public SagaOrquestador(List<PasoSaga> pasos) { this.pasos = pasos; }

    public void ejecutar(Map<String,Object> ctx) {
        for (PasoSaga paso : pasos) {
            try {
                System.out.println("▶ " + paso.getNombre());
                paso.ejecutar(ctx);
                ejecutados.push(paso);
            } catch (Exception e) {
                System.out.println("❌ Falló: " + e.getMessage());
                ejecutados.forEach(p -> p.compensar(ctx));
                throw new RuntimeException("Saga revertida", e);
            }
        }
        System.out.println("✅ Saga completada");
    }
}

🚫 Cuándo NO usar

  • Para transacciones simples que afectan a un único servicio o base de datos; usa una transacción ACID normal.
  • Cuando la consistencia eventual no es aceptable y se requiere atomicidad estricta.
  • Si las compensaciones son imposibles o extremadamente difíciles (ej: ya se envió un email).
