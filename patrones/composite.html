Composite

Componer objetos en estructuras de árbol para representar jerarquías parte-todo y tratar objetos individuales y compuestos de manera uniforme.

Estructural

📖 Descripción

Composite permite tratar un objeto individual y una colección de objetos de la misma manera a través de una interfaz común. Los objetos 'hoja' realizan el trabajo real; los objetos 'compuestos' delegan a sus hijos. Ideal para estructuras recursivas como sistemas de archivos, menús anidados o árboles de widgets.

🎯 Casos de uso prácticos

1. Sistema de archivos

Carpetas y archivos comparten la operación `calcularTamano()`, una carpeta suma el tamaño de sus hijos.

2. Árbol de menú UI

Items de menú simples y submenús anidados se renderizan con la misma interfaz.

💻 Ejemplos de código

JavaScript
// Componente
class ElementoSistema {
  constructor(nombre) { this.nombre = nombre; }
  calcularTamano() { throw new Error('Abstracto'); }
  mostrar(indent = '') { throw new Error('Abstracto'); }
}

// Hoja
class Archivo extends ElementoSistema {
  constructor(nombre, tamano) { super(nombre); this.tamano = tamano; }
  calcularTamano() { return this.tamano; }
  mostrar(indent = '') { console.log(`${indent}📄 ${this.nombre} (${this.tamano}KB)`); }
}

// Compuesto
class Carpeta extends ElementoSistema {
  constructor(nombre) { super(nombre); this.hijos = []; }
  agregar(elemento) { this.hijos.push(elemento); }
  calcularTamano() { return this.hijos.reduce((s, h) => s + h.calcularTamano(), 0); }
  mostrar(indent = '') {
    console.log(`${indent}📁 ${this.nombre}`);
    this.hijos.forEach(h => h.mostrar(indent + '  '));
  }
}

// Uso
const raiz = new Carpeta('proyecto');
const src  = new Carpeta('src');
src.agregar(new Archivo('index.js', 12));
src.agregar(new Archivo('app.js', 8));
raiz.agregar(src);
raiz.agregar(new Archivo('README.md', 2));
raiz.mostrar();
console.log('Total:', raiz.calcularTamano(), 'KB');
Java
public abstract class ElementoSistema {
    protected String nombre;
    public ElementoSistema(String nombre) { this.nombre = nombre; }
    public abstract int calcularTamano();
    public abstract void mostrar(String indent);
}

public class Archivo extends ElementoSistema {
    private int tamano;
    public Archivo(String nombre, int tamano) {
        super(nombre); this.tamano = tamano;
    }
    public int calcularTamano() { return tamano; }
    public void mostrar(String indent) {
        System.out.println(indent + "📄 " + nombre + " (" + tamano + "KB)");
    }
}

public class Carpeta extends ElementoSistema {
    private List<ElementoSistema> hijos = new ArrayList<>();
    public Carpeta(String nombre) { super(nombre); }
    public void agregar(ElementoSistema e) { hijos.add(e); }
    public int calcularTamano() {
        return hijos.stream().mapToInt(ElementoSistema::calcularTamano).sum();
    }
    public void mostrar(String indent) {
        System.out.println(indent + "📁 " + nombre);
        hijos.forEach(h -> h.mostrar(indent + "  "));
    }
}

🚫 Cuándo NO usar

  • Cuando la jerarquía nunca tendrá más de un nivel; una simple lista es suficiente.
  • Si los componentes hoja y compuesto tienen operaciones muy distintas, forzar una interfaz común resulta artificial.
  • Para estructuras planas donde la recursividad añade overhead sin beneficio real.
