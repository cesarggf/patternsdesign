Dead Letter Channel

Redirigir automáticamente los mensajes que no pueden procesarse correctamente a un canal especial para su inspección y manejo posterior.

Integración

📖 Descripción

El Dead Letter Channel (DLC) es un canal de destino para mensajes fallidos: mensajes con formato inválido, mensajes expirados o aquellos que causaron errores después de N reintentos. Evita que mensajes problemáticos bloqueen el canal principal. Los operadores pueden inspeccionar el DLC, corregir los mensajes y reenviarlos.

🎯 Casos de uso prácticos

1. Procesamiento de facturas

Facturas con esquema inválido van al DLC; el equipo las corrige manualmente y las reinyecta.

2. Cola de emails fallidos

Emails que no pudieron enviarse tras 3 reintentos se mueven al DLC con el motivo del fallo.

💻 Ejemplos de código

JavaScript
class ColaMensajes {
  constructor(nombre, opciones = {}) {
    this.nombre = nombre;
    this.cola = [];
    this.maxReintentos = opciones.maxReintentos || 3;
    this.dlc = opciones.dlc || null;
  }

  encolar(mensaje) {
    this.cola.push({ ...mensaje, intentos: 0 });
  }

  procesar(handler) {
    while (this.cola.length > 0) {
      const msg = this.cola.shift();
      try {
        handler(msg);
        console.log(`✅ [${this.nombre}] procesado: ${msg.id}`);
      } catch (err) {
        msg.intentos++;
        msg.ultimoError = err.message;
        if (msg.intentos < this.maxReintentos) {
          console.log(`🔄 Reintento ${msg.intentos}/${this.maxReintentos}: ${msg.id}`);
          this.cola.push(msg);
        } else if (this.dlc) {
          console.log(`💀 Enviando a DLC: ${msg.id} — ${err.message}`);
          this.dlc.encolar(msg);
        }
      }
    }
  }
}

const dlc  = new ColaMensajes('dead-letter');
const cola = new ColaMensajes('facturas', { maxReintentos: 2, dlc });

cola.encolar({ id: 'F-001', monto: 100 });
cola.encolar({ id: 'F-002', monto: null }); // inválida

cola.procesar(msg => {
  if (!msg.monto) throw new Error('Monto requerido');
  console.log(`Factura ${msg.id}: €${msg.monto}`);
});

console.log('\nDead Letter Channel:', dlc.cola);
Java
public class ColaMensajes {
    private Queue<Mensaje> cola = new LinkedList<>();
    private int maxReintentos;
    private ColaMensajes dlc;

    public ColaMensajes(int maxReintentos, ColaMensajes dlc) {
        this.maxReintentos = maxReintentos; this.dlc = dlc;
    }

    public void encolar(Mensaje m) { cola.offer(m); }

    public void procesar(Consumer<Mensaje> handler) {
        while (!cola.isEmpty()) {
            Mensaje msg = cola.poll();
            try {
                handler.accept(msg);
                System.out.println("✅ Procesado: " + msg.id);
            } catch (Exception e) {
                msg.intentos++;
                if (msg.intentos < maxReintentos) {
                    cola.offer(msg);
                } else if (dlc != null) {
                    msg.error = e.getMessage();
                    System.out.println("💀 DLC: " + msg.id);
                    dlc.encolar(msg);
                }
            }
        }
    }
}

  • Spring AMQP
  • Apache Kafka (retry topics)

🚫 Cuándo NO usar

  • Cuando los errores deben tratarse sincrónicamente con el flujo principal.
  • Si el DLC no va a ser monitoreado activamente, los mensajes muertos se acumularán sin resolución.
  • Para errores transitorios (red, DB temporalmente caída); usa reintentos con backoff antes del DLC.
