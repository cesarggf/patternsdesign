MVC (Model-View-Controller)

Separar una aplicación en tres componentes — Modelo, Vista y Controlador — para desacoplar la lógica de negocio, la presentación y la gestión de entradas.

Arquitectura

📖 Descripción

En MVC el Modelo gestiona los datos y la lógica de negocio; la Vista muestra la información al usuario; el Controlador intercepta las entradas del usuario, actualiza el Modelo y selecciona la Vista adecuada. Las variantes modernas separan aún más las responsabilidades en MVP y MVVM.

🎯 Casos de uso prácticos

1. Aplicación web de gestión de tareas

El Modelo gestiona las tareas en BD, el Controlador procesa las peticiones HTTP, las Vistas renderizan el HTML.

2. Aplicación de escritorio

Un editor de texto: el Modelo es el documento, la Vista es el área de texto, el Controlador maneja teclas y menús.

💻 Ejemplos de código

JavaScript
// Modelo
class ModeloTareas {
  constructor() { this.tareas = []; this.listeners = []; }
  agregar(texto) {
    this.tareas.push({ id: Date.now(), texto, completada: false });
    this.notificar();
  }
  completar(id) {
    const t = this.tareas.find(t => t.id === id);
    if (t) { t.completada = true; this.notificar(); }
  }
  onChange(fn) { this.listeners.push(fn); }
  notificar() { this.listeners.forEach(fn => fn(this.tareas)); }
}

// Vista
class VistaTareas {
  renderizar(tareas) {
    console.log('=== Mis Tareas ===');
    tareas.forEach(t =>
      console.log(`[${t.completada ? '✓' : ' '}] ${t.id}: ${t.texto}`));
  }
  mostrarError(msg) { console.error('Error:', msg); }
}

// Controlador
class ControladorTareas {
  constructor(modelo, vista) {
    this.modelo = modelo;
    this.vista  = vista;
    modelo.onChange(tareas => vista.renderizar(tareas));
  }
  agregarTarea(texto) {
    if (!texto.trim()) return this.vista.mostrarError('El texto no puede estar vacío');
    this.modelo.agregar(texto);
  }
  completarTarea(id) { this.modelo.completar(id); }
}

const modelo = new ModeloTareas();
const vista  = new VistaTareas();
const ctrl   = new ControladorTareas(modelo, vista);

ctrl.agregarTarea('Estudiar patrones de diseño');
ctrl.agregarTarea('Practicar con ejemplos');
ctrl.completarTarea(modelo.tareas[0].id);
Java
// Modelo
public class Tarea {
    public int id; public String texto; public boolean completada;
    public Tarea(int id, String texto) { this.id=id; this.texto=texto; }
}

public class ModeloTareas {
    private List<Tarea> tareas = new ArrayList<>();
    private List<Runnable> listeners = new ArrayList<>();
    private int nextId = 1;

    public void agregar(String texto) {
        tareas.add(new Tarea(nextId++, texto));
        listeners.forEach(Runnable::run);
    }
    public List<Tarea> getTareas() { return tareas; }
    public void onChange(Runnable fn) { listeners.add(fn); }
}

// Vista
public class VistaTareas {
    public void renderizar(List<Tarea> tareas) {
        System.out.println("=== Mis Tareas ===");
        tareas.forEach(t ->
            System.out.println("[" + (t.completada?"✓":" ") + "] " + t.texto));
    }
}

// Controlador
public class ControladorTareas {
    private ModeloTareas modelo; private VistaTareas vista;
    public ControladorTareas(ModeloTareas m, VistaTareas v) {
        modelo=m; vista=v;
        m.onChange(() -> v.renderizar(m.getTareas()));
    }
    public void agregarTarea(String texto) { modelo.agregar(texto); }
}

🚫 Cuándo NO usar

  • Para aplicaciones muy simples con una sola pantalla y sin lógica de negocio.
  • Cuando el Controlador acumula demasiada lógica y se convierte en un 'fat controller'; usa MVVM o Clean Architecture.
  • En interfaces con estado complejo y reactivo donde frameworks como React con Redux son más naturales.
