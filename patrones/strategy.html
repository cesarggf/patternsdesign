Strategy

Definir una familia de algoritmos, encapsular cada uno y hacerlos intercambiables, permitiendo que el algoritmo varíe independientemente del cliente.

Comportamiento

📖 Descripción

Strategy extrae los algoritmos variantes en clases separadas llamadas estrategias. El contexto mantiene una referencia a una estrategia y la delega el trabajo. En tiempo de ejecución se puede cambiar la estrategia sin modificar el contexto. Elimina condicionales extensos para selección de algoritmos.

🎯 Casos de uso prácticos

1. Ordenamiento configurable

Un sistema puede ordenar datos por distintos criterios (precio, fecha, popularidad) sin cambiar el código base.

2. Cálculo de descuentos

Distintas estrategias: porcentual, monto fijo, 2x1 — aplicadas según el tipo de cliente.

💻 Ejemplos de código

JavaScript
// Estrategias de descuento
const descuentoPorcentaje = (precio, valor) => precio * (1 - valor / 100);
const descuentoFijo       = (precio, valor) => Math.max(0, precio - valor);
const descuento2x1        = (precio)        => precio / 2;

// Contexto
class Carrito {
  constructor() { this.items = []; this.estrategia = null; }

  setEstrategia(fn) { this.estrategia = fn; }
  agregar(item)     { this.items.push(item); }

  calcularTotal() {
    const subtotal = this.items.reduce((s, i) => s + i.precio, 0);
    return this.estrategia ? this.estrategia(subtotal, 20) : subtotal;
  }
}

const carrito = new Carrito();
carrito.agregar({ nombre: 'Laptop', precio: 1000 });
carrito.agregar({ nombre: 'Mouse',  precio: 50 });

carrito.setEstrategia(descuentoPorcentaje);
console.log('Con 20% dto:', carrito.calcularTotal()); // 840

carrito.setEstrategia(descuentoFijo);
console.log('Con $20 dto:', carrito.calcularTotal()); // 1030

carrito.setEstrategia(descuento2x1);
console.log('Con 2x1:',    carrito.calcularTotal()); // 525
Java
@FunctionalInterface
public interface EstrategiaDescuento {
    double aplicar(double precio, double parametro);
}

public class Carrito {
    private List<Double> precios = new ArrayList<>();
    private EstrategiaDescuento estrategia;

    public void setEstrategia(EstrategiaDescuento e) { estrategia = e; }
    public void agregar(double precio) { precios.add(precio); }

    public double calcularTotal() {
        double subtotal = precios.stream().mapToDouble(Double::doubleValue).sum();
        return estrategia != null ? estrategia.aplicar(subtotal, 20) : subtotal;
    }

    public static void main(String[] args) {
        Carrito c = new Carrito();
        c.agregar(1000.0); c.agregar(50.0);

        c.setEstrategia((precio, val) -> precio * (1 - val / 100)); // 20% dto
        System.out.println("Con 20% dto: " + c.calcularTotal());    // 840.0

        c.setEstrategia((precio, val) -> Math.max(0, precio - val)); // fijo
        System.out.println("Con $20 dto: " + c.calcularTotal());    // 1030.0
    }
}

🚫 Cuándo NO usar

  • Cuando solo hay una variante del algoritmo y no se espera que cambie.
  • Si los algoritmos son tan simples que extraerlos a clases completas es excesivo; bastan funciones.
  • Cuando los clientes necesitan conocer las diferencias entre estrategias para elegir correctamente.

🛠️ Software que lo implementa

  • Java Comparator (Collections.sort)

  • javax.crypto.Cipher (distintos algoritmos)
  • Jackson ObjectMapper (serialización configurable)
