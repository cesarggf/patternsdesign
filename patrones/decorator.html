Decorator

Añadir responsabilidades a un objeto dinámicamente, como alternativa flexible a la herencia para extender funcionalidad.

Estructural

📖 Descripción

Decorator envuelve un objeto existente para añadirle comportamiento antes o después de delegar la llamada al objeto envuelto. Varios decoradores pueden apilarse como capas de cebolla. Difiere de la herencia en que la extensión ocurre en tiempo de ejecución y puede combinarse libremente.

🎯 Casos de uso prácticos

1. Streams de I/O

Apilar compresión + cifrado + buffering sobre un stream base de escritura a archivo.

2. Middleware HTTP

Añadir autenticación, logging y rate-limiting a un handler HTTP base de forma componible.

💻 Ejemplos de código

JavaScript
// Componente base
class Cafe {
  costo() { return 1.0; }
  descripcion() { return 'Café'; }
}

// Decorador base
class DecoradorCafe {
  constructor(cafe) { this.cafe = cafe; }
  costo() { return this.cafe.costo(); }
  descripcion() { return this.cafe.descripcion(); }
}

// Decoradores concretos
class ConLeche extends DecoradorCafe {
  costo()       { return this.cafe.costo() + 0.5; }
  descripcion() { return this.cafe.descripcion() + ', leche'; }
}
class ConVainilla extends DecoradorCafe {
  costo()       { return this.cafe.costo() + 0.7; }
  descripcion() { return this.cafe.descripcion() + ', vainilla'; }
}
class ConChocolate extends DecoradorCafe {
  costo()       { return this.cafe.costo() + 0.9; }
  descripcion() { return this.cafe.descripcion() + ', chocolate'; }
}

// Uso — combinación dinámica
let pedido = new Cafe();
pedido = new ConLeche(pedido);
pedido = new ConVainilla(pedido);
pedido = new ConChocolate(pedido);

console.log(pedido.descripcion()); // Café, leche, vainilla, chocolate
console.log(pedido.costo());       // 3.1
Java
public interface Cafe {
    double costo();
    String descripcion();
}

public class CafeSimple implements Cafe {
    public double costo()       { return 1.0; }
    public String descripcion() { return "Café"; }
}

public abstract class DecoradorCafe implements Cafe {
    protected Cafe cafe;
    public DecoradorCafe(Cafe cafe) { this.cafe = cafe; }
}

public class ConLeche extends DecoradorCafe {
    public ConLeche(Cafe cafe) { super(cafe); }
    public double costo()       { return cafe.costo() + 0.5; }
    public String descripcion() { return cafe.descripcion() + ", leche"; }
}

public class ConVainilla extends DecoradorCafe {
    public ConVainilla(Cafe cafe) { super(cafe); }
    public double costo()       { return cafe.costo() + 0.7; }
    public String descripcion() { return cafe.descripcion() + ", vainilla"; }
}

// Uso
Cafe pedido = new ConVainilla(new ConLeche(new CafeSimple()));
System.out.println(pedido.descripcion() + " = " + pedido.costo());

🚫 Cuándo NO usar

  • Cuando el orden de los decoradores importa y no se gestiona correctamente, causa bugs difíciles de rastrear.
  • Demasiados decoradores anidados complican el debugging (stack de llamadas profundo).
  • Si la funcionalidad extra siempre aplica al 100% de los casos, simplemente extiende la clase.
