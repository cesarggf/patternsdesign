Command

Encapsular una solicitud como objeto, permitiendo parametrizar clientes, hacer cola de solicitudes, registrar operaciones y soportar operaciones deshechas (undo).

Comportamiento

📖 Descripción

Command convierte acciones en objetos independientes. El Invocador almacena y ejecuta comandos sin conocer su implementación. El Receptor realiza el trabajo real. Esta separación permite implementar undo/redo, colas de operaciones, logging de acciones y transacciones.

🎯 Casos de uso prácticos

1. Editor de texto con Undo/Redo

Cada acción de escritura o borrado es un comando; el historial permite deshacer y rehacer.

2. Sistema de macros

Grabar una secuencia de comandos del usuario y reproducirlos automáticamente.

💻 Ejemplos de código

JavaScript
// Receptor
class Editor {
  constructor() { this.texto = ''; }
  insertar(texto) { this.texto += texto; }
  borrar(n)       { this.texto = this.texto.slice(0, -n); }
}

// Interfaz Command
class Comando {
  ejecutar() {}
  deshacer() {}
}

// Comandos concretos
class ComandoInsertar extends Comando {
  constructor(editor, texto) { super(); this.editor = editor; this.texto = texto; }
  ejecutar() { this.editor.insertar(this.texto); }
  deshacer() { this.editor.borrar(this.texto.length); }
}

// Invocador con historial
class Historial {
  constructor() { this.pila = []; }
  ejecutar(cmd)  { cmd.ejecutar(); this.pila.push(cmd); }
  deshacer()     { this.pila.pop()?.deshacer(); }
}

const editor    = new Editor();
const historial = new Historial();

historial.ejecutar(new ComandoInsertar(editor, 'Hola'));
historial.ejecutar(new ComandoInsertar(editor, ' mundo'));
console.log(editor.texto); // Hola mundo

historial.deshacer();
console.log(editor.texto); // Hola
Java
public interface Comando {
    void ejecutar();
    void deshacer();
}

public class Editor {
    private StringBuilder texto = new StringBuilder();
    public void insertar(String t) { texto.append(t); }
    public void borrar(int n) { texto.delete(texto.length()-n, texto.length()); }
    public String getText() { return texto.toString(); }
}

public class ComandoInsertar implements Comando {
    private Editor editor;
    private String texto;
    public ComandoInsertar(Editor e, String t) { editor=e; texto=t; }
    public void ejecutar() { editor.insertar(texto); }
    public void deshacer() { editor.borrar(texto.length()); }
}

public class Historial {
    private Deque<Comando> pila = new ArrayDeque<>();
    public void ejecutar(Comando cmd) { cmd.ejecutar(); pila.push(cmd); }
    public void deshacer() { if (!pila.isEmpty()) pila.pop().deshacer(); }
}

🚫 Cuándo NO usar

  • Para operaciones simples sin necesidad de undo, historial ni colas; agrega burocracia innecesaria.
  • Cuando las operaciones siempre se ejecutan de inmediato y no se reutilizan.
  • Si el sistema ya tiene un mecanismo transaccional robusto que hace lo mismo.
