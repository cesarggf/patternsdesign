Observer

Definir una dependencia uno-a-muchos entre objetos para que cuando uno cambie su estado, todos sus dependientes sean notificados automáticamente.

Comportamiento

📖 Descripción

Observer (también llamado Publicar-Suscribir) permite que un objeto (Subject/Observable) mantenga una lista de observadores y les notifique cambios automáticamente. Los observadores se suscriben y desuscriben dinámicamente. Es la base de la programación reactiva y los sistemas de eventos.

🎯 Casos de uso prácticos

1. Sistema de eventos de UI

Un componente de precio actualiza gráficas, alertas y resúmenes cuando cambia el valor.

2. Stock de inventario

Cuando el stock de un producto baja del mínimo, notifica al sistema de compras y al dashboard.

💻 Ejemplos de código

JavaScript
class EventEmitter {
  constructor() { this.oyentes = {}; }

  suscribir(evento, fn) {
    if (!this.oyentes[evento]) this.oyentes[evento] = [];
    this.oyentes[evento].push(fn);
  }
  desuscribir(evento, fn) {
    this.oyentes[evento] = (this.oyentes[evento] || []).filter(f => f !== fn);
  }
  emitir(evento, datos) {
    (this.oyentes[evento] || []).forEach(fn => fn(datos));
  }
}

class Inventario extends EventEmitter {
  constructor() {
    super();
    this.stock = {};
  }
  actualizar(producto, cantidad) {
    this.stock[producto] = cantidad;
    this.emitir('cambio', { producto, cantidad });
    if (cantidad < 5) this.emitir('stockBajo', { producto, cantidad });
  }
}

const inv = new Inventario();
inv.suscribir('cambio',   ({ producto, cantidad }) =>
  console.log(`📦 ${producto}: ${cantidad} uds`));
inv.suscribir('stockBajo', ({ producto }) =>
  console.log(`⚠️ Stock crítico: ${producto} — solicitar reposición`));

inv.actualizar('Laptop', 10);
inv.actualizar('Mouse', 3);
Java
public interface Observador {
    void actualizar(String producto, int cantidad);
}

public class Inventario {
    private Map<String, Integer> stock = new HashMap<>();
    private List<Observador> observadores = new ArrayList<>();

    public void suscribir(Observador o)   { observadores.add(o); }
    public void desuscribir(Observador o) { observadores.remove(o); }

    private void notificar(String p, int c) {
        observadores.forEach(o -> o.actualizar(p, c));
    }

    public void actualizar(String producto, int cantidad) {
        stock.put(producto, cantidad);
        notificar(producto, cantidad);
    }
}

// Observadores concretos
public class AlertaStock implements Observador {
    public void actualizar(String p, int c) {
        if (c < 5) System.out.println("⚠️ Stock crítico: " + p);
    }
}
public class Dashboard implements Observador {
    public void actualizar(String p, int c) {
        System.out.println("📊 Dashboard: " + p + " = " + c);
    }
}

🚫 Cuándo NO usar

  • Cuando las notificaciones en cascada son impredecibles y difíciles de depurar.
  • Si el orden de notificación importa y no está garantizado, pueden surgir condiciones de carrera.
  • Para observadores que rara vez reciben notificaciones relevantes, el polling puede ser más eficiente.
