Visitor

Definir una operación a realizar sobre los elementos de una estructura de objetos sin cambiar las clases de dichos elementos.

Comportamiento

📖 Descripción

Visitor permite añadir nuevas operaciones a una jerarquía de clases sin modificarlas, usando el mecanismo de double dispatch. Cada elemento acepta un visitante; el visitante tiene un método `visitar` sobrecargado para cada tipo de elemento. Ideal cuando la jerarquía es estable pero las operaciones cambian frecuentemente.

🎯 Casos de uso prácticos

1. Generador de reportes

Visitar elementos de una factura (productos, descuentos, impuestos) para generar texto o HTML.

2. Compilador — análisis semántico

Distintos visitantes (verificador de tipos, generador de código) recorren el árbol sintáctico.

💻 Ejemplos de código

JavaScript
// Elementos
class Producto {
  constructor(nombre, precio) { this.nombre = nombre; this.precio = precio; }
  aceptar(visitante) { visitante.visitarProducto(this); }
}
class Descuento {
  constructor(porcentaje) { this.porcentaje = porcentaje; }
  aceptar(visitante) { visitante.visitarDescuento(this); }
}
class Impuesto {
  constructor(tasa) { this.tasa = tasa; }
  aceptar(visitante) { visitante.visitarImpuesto(this); }
}

// Visitantes
class CalculadorTotal {
  constructor() { this.total = 0; this.base = 0; }
  visitarProducto(p) { this.base += p.precio; this.total += p.precio; }
  visitarDescuento(d) { this.total -= this.base * (d.porcentaje / 100); }
  visitarImpuesto(i)  { this.total += this.base * (i.tasa / 100); }
}

class GeneradorHTML {
  constructor() { this.html = ''; }
  visitarProducto(p) { this.html += `<li>${p.nombre}: €${p.precio}</li>`; }
  visitarDescuento(d) { this.html += `<li class="desc">Descuento: -${d.porcentaje}%</li>`; }
  visitarImpuesto(i)  { this.html += `<li class="tax">IVA: +${i.tasa}%</li>`; }
}

const factura = [new Producto('Laptop',1000), new Descuento(10), new Impuesto(21)];
const calc = new CalculadorTotal();
factura.forEach(e => e.aceptar(calc));
console.log('Total:', calc.total.toFixed(2)); // 1110.00
Java
public interface Visitante {
    void visitarProducto(Producto p);
    void visitarDescuento(Descuento d);
    void visitarImpuesto(Impuesto i);
}

public interface ElementoFactura {
    void aceptar(Visitante v);
}

public class Producto implements ElementoFactura {
    public String nombre; public double precio;
    public Producto(String n, double p) { nombre=n; precio=p; }
    public void aceptar(Visitante v) { v.visitarProducto(this); }
}

public class CalculadorTotal implements Visitante {
    private double total=0, base=0;
    public void visitarProducto(Producto p) { base+=p.precio; total+=p.precio; }
    public void visitarDescuento(Descuento d) { total -= base*(d.porcentaje/100); }
    public void visitarImpuesto(Impuesto i)   { total += base*(i.tasa/100); }
    public double getTotal() { return total; }
}

🚫 Cuándo NO usar

  • Cuando la jerarquía de objetos cambia frecuentemente; añadir un nuevo elemento requiere modificar todos los visitantes.
  • Si el encapsulamiento es crítico; los visitantes necesitan acceder a los detalles internos de los elementos.
  • Para jerarquías simples con pocas operaciones, la complejidad del double dispatch no se justifica.
