Proxy

Proporcionar un sustituto o marcador de posición de otro objeto para controlar el acceso a él.

Estructural

📖 Descripción

Proxy actúa como intermediario entre el cliente y el objeto real. Puede añadir funcionalidad como carga diferida (lazy loading), control de acceso, logging, caché o sincronización, sin que el cliente lo perciba. El Proxy implementa la misma interfaz que el objeto real.

🎯 Casos de uso prácticos

1. Caché de imágenes

Un proxy carga la imagen del disco solo la primera vez; las siguientes llamadas devuelven la versión en caché.

2. Control de acceso a API

Un proxy verifica que el usuario tenga permisos antes de llamar al servicio real.

💻 Ejemplos de código

JavaScript
// Interfaz común
class IServicioUsuarios {
  obtener(id) { throw new Error('Abstracto'); }
}

// Servicio real (costoso)
class ServicioUsuarios extends IServicioUsuarios {
  obtener(id) {
    console.log(`🔍 DB: consultando usuario ${id}`);
    return { id, nombre: 'Cesar', rol: 'admin' };
  }
}

// Proxy con caché + control de acceso
class ProxyUsuarios extends IServicioUsuarios {
  constructor() {
    super();
    this.servicio = new ServicioUsuarios();
    this.cache = {};
    this.usuarioActual = null;
  }
  autenticar(rol) { this.usuarioActual = rol; }

  obtener(id) {
    if (this.usuarioActual !== 'admin') {
      throw new Error('Acceso denegado');
    }
    if (this.cache[id]) {
      console.log(`⚡ Cache: usuario ${id}`);
      return this.cache[id];
    }
    const usuario = this.servicio.obtener(id);
    this.cache[id] = usuario;
    return usuario;
  }
}

const proxy = new ProxyUsuarios();
proxy.autenticar('admin');
proxy.obtener(1); // DB
proxy.obtener(1); // Cache
Java
public interface IServicioUsuarios {
    Usuario obtener(int id);
}

public class ServicioUsuarios implements IServicioUsuarios {
    public Usuario obtener(int id) {
        System.out.println("DB: consultando usuario " + id);
        return new Usuario(id, "Cesar");
    }
}

public class ProxyUsuarios implements IServicioUsuarios {
    private ServicioUsuarios servicio = new ServicioUsuarios();
    private Map<Integer, Usuario> cache = new HashMap<>();
    private String rolActual;

    public void autenticar(String rol) { this.rolActual = rol; }

    public Usuario obtener(int id) {
        if (!"admin".equals(rolActual)) throw new SecurityException("Acceso denegado");
        return cache.computeIfAbsent(id, k -> {
            System.out.println("Proxy: delegando a servicio real");
            return servicio.obtener(k);
        });
    }
}

🚫 Cuándo NO usar

  • Cuando el objeto es barato de crear y no requiere control de acceso ni caché.
  • Si el Proxy introduce latencia perceptible sin un beneficio concreto.
  • Para lógica de negocio compleja, Decorator o Middleware son más apropiados.
