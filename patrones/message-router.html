Message Router

Encaminar mensajes a distintos canales de destino según el contenido o propiedades del mensaje, sin que el productor conozca los destinos.

Integración

📖 Descripción

El Message Router inspecciona cada mensaje y decide a qué canal enviarlo. Desacopla al emisor de los receptores concretos. Variantes comunes: Content-Based Router (por contenido), Recipient List (a múltiples destinos), Splitter (divide un mensaje en partes) y Aggregator (combina varios mensajes en uno).

🎯 Casos de uso prácticos

1. Enrutamiento de pagos

Un router dirige pagos a Stripe, PayPal o transferencia bancaria según el método elegido por el usuario.

2. Clasificación de alertas

Alertas de monitoreo se enrutan a PagerDuty (crítico), Slack (warning) o se descartan (info).

💻 Ejemplos de código

JavaScript
class RouterMensajes {
  constructor() { this.rutas = []; this.defaultFn = null; }

  cuando(condicion, handler) { this.rutas.push({ condicion, handler }); return this; }
  porDefecto(handler) { this.defaultFn = handler; return this; }

  enrutar(mensaje) {
    const ruta = this.rutas.find(r => r.condicion(mensaje));
    if (ruta) return ruta.handler(mensaje);
    if (this.defaultFn) return this.defaultFn(mensaje);
    console.warn('Sin ruta para:', mensaje);
  }
}

// Tipos de pago
const router = new RouterMensajes()
  .cuando(m => m.metodo === 'tarjeta',    m => console.log(`[Stripe] procesando $${m.monto}`))
  .cuando(m => m.metodo === 'paypal',     m => console.log(`[PayPal] procesando $${m.monto}`))
  .cuando(m => m.metodo === 'transferencia', m => console.log(`[Banco] procesando $${m.monto}`))
  .porDefecto(m => console.log(`[Error] Método desconocido: ${m.metodo}`));

router.enrutar({ metodo: 'tarjeta',      monto: 99.99 });
router.enrutar({ metodo: 'paypal',       monto: 49.00 });
router.enrutar({ metodo: 'criptomoneda', monto: 0.01 });
Java
import java.util.*;
import java.util.function.*;

public class RouterMensajes<T> {
    record Ruta<T>(Predicate<T> condicion, Consumer<T> handler) {}
    private List<Ruta<T>> rutas = new ArrayList<>();
    private Consumer<T> defaultHandler;

    public RouterMensajes<T> cuando(Predicate<T> cond, Consumer<T> handler) {
        rutas.add(new Ruta<>(cond, handler)); return this;
    }
    public RouterMensajes<T> porDefecto(Consumer<T> h) { defaultHandler = h; return this; }

    public void enrutar(T mensaje) {
        rutas.stream().filter(r -> r.condicion().test(mensaje)).findFirst()
             .ifPresentOrElse(r -> r.handler().accept(mensaje),
                              () -> { if (defaultHandler != null) defaultHandler.accept(mensaje); });
    }
}

🚫 Cuándo NO usar

  • Cuando la lógica de enrutamiento es tan compleja que el router se convierte en un bottleneck.
  • Si los destinos cambian con tanta frecuencia que el router requiere redespliegues constantes; usa reglas externalizadas.
  • Para sistemas simples con un único destino fijo.
